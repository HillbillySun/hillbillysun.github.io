Project 1 - Q2 生存分析：一份从“谁会流失”到“如何留存”的数据报告

对应课程要求 Q2(2)：完整记录生存分析过程与结果，并以博客形式发布（Q2(4)）。

1. 业务背景：为什么要做这件事？

电信业务的核心问题不是“有没有流失”，而是“客户会在什么时候流失、哪些人更容易提前流失、我们应该优先保住谁”。 因此我没有只做普通分类，而是采用生存分析来刻画客户留存随时间的变化，再把结果转化为可执行的运营建议。

2. 分析目标：要回答的三个问题

  • 客户总体留存周期有多长？
  • 不同服务类型下，流失风险是否显著不同？
  • 把留存曲线转化为价值后，客户在 12/24/36 月分别值多少钱？

3. 数据与方法

数据来自 Telco churn 场景，先完成清洗与建模字段准备，再按“描述 - 解释 - 价值”三层推进：

  • 分析数据集（Silver）规模：3351
  • 描述层：Kaplan-Meier（总体与分组生存曲线）
  • 解释层：Cox PH 与 AFT（识别风险驱动因素）
  • 价值层：CLV / NPV（把生存概率转成经营价值）
  • 实现文件：Q2_survival_analysis_rewrite.ipynb

4. 分析过程（从现象到解释）

  • 数据清洗与建模字段准备（含时间与事件定义）
  • 总体与分组 Kaplan-Meier 曲线及 log-rank 检验
  • Cox 比例风险模型拟合与风险比解释
  • AFT 模型作为对照时间建模
  • 基于生存曲线进行 CLV 累计 NPV 估计

5. 核心结果（可直接用于决策）

  • KM 中位生存时间：34.0 个月
  • DSL vs Fiber optic 的 log-rank：p = 5.241449e-07
  • Cox concordance：0.64
  • CLV 累计 NPV（12/24/36 月）：320.31 / 589.49 / 818.84

这组结果给出一个清晰信号：客户留存并非线性下降，不同服务分组之间存在显著差异， 且这种差异会持续放大到长期价值（CLV）上。

5.1 先看现象：Kaplan-Meier 曲线

从分组曲线与 log-rank 结果可以确认：不同服务组的留存路径并不一致， 这说明后续做统一运营策略会损失效果，应该做分层运营。

5.2 再看解释：Cox / AFT / CLV 结果

Cox/AFT 帮助定位“风险高低”，CLV 则回答“值不值得投入留存成本”。 两者结合后，策略不再是“谁容易流失就救谁”，而是“优先保住高风险且高价值人群”。

6. 工程复盘：Text-to-SQL 的失败与修复

在 MySQL + LLM 环节，出现了三个高频问题：分位函数方言不兼容、加权中位数实现错误、MAD 双层中位数失败。 我最终都改写为 MySQL 可执行窗口函数版本，确保从“提问”到“可运行 SQL”形成闭环。

7. 结论与行动建议

  • 建立“分层留存”而非“一刀切”策略，按服务类型与风险等级分组运营。
  • 将 Cox/AFT 风险评分与 CLV 联合使用，优先资源投向“高风险 + 高价值”客户。
  • 将当前流程沉淀为月度监控看板：生存曲线、关键风险因子、分层 CLV 同步更新。
